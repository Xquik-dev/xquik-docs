The Xquik MCP server exposes 26 tools that AI agents can use to manage monitors, query events, search tweets, look up users and tweets, check follow relationships, run extractions and draws, configure webhooks, check account status, get trending topics, manage subscriptions, and compose algorithm-optimized tweets. Connect to the MCP server at `https://xquik.com/mcp` with your API key.

## Monitoring

### list-monitors

List all monitored X accounts for the authenticated user.

**Input:** None

**Example:**

> **Prompt:** "Show me all my active monitors"

```json Tool Call
{
  "name": "list-monitors",
  "arguments": {}
}
```

```json Result
{
  "monitors": [
    {
      "id": "7",
      "xUsername": "elonmusk",
      "eventTypes": ["tweet.new", "tweet.reply"],
      "isActive": true,
      "createdAt": "2026-02-24T10:30:00.000Z"
    },
    {
      "id": "12",
      "xUsername": "OpenAI",
      "eventTypes": ["tweet.new"],
      "isActive": true,
      "createdAt": "2026-02-20T08:00:00.000Z"
    }
  ]
}
```

```json Tool Call
    "eventTypes": ["tweet.new", "tweet.reply"]
  }
}
```

```json Result
  "xUsername": "elonmusk",
  "eventTypes": ["tweet.new", "tweet.reply"],
  "isActive": true,
  "createdAt": "2026-02-24T10:30:00.000Z"
```

## Events

### get-events

Query events from monitored accounts with optional filters and cursor-based pagination.

**Input:**

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `limit` | number | No | 50 | Results per page (max 100) |
| `monitorId` | string | No | - | Filter by monitor ID |
| `eventType` | string | No | - | Filter by event type |
| `afterCursor` | string | No | - | Pagination cursor from previous response |

**Example:**

> **Prompt:** "Get the latest 10 tweets from monitor 7"

```json Tool Call
{
  "name": "get-events",
  "arguments": {
    "limit": 10,
    "monitorId": "7",
    "eventType": "tweet.new"
  }
}
```

```json Result
{
  "events": [
    {
      "id": "9001",
      "eventType": "tweet.new",
      "monitoredAccountId": "7",
      "xUsername": "elonmusk",
      "occurredAt": "2026-02-24T14:22:00.000Z",
      "eventData": {
        "tweetId": "1893456789012345678",
        "text": "The future is now.",
        "metrics": {
          "likes": 42000,
          "retweets": 8500,
          "replies": 3200
        }
      },
      "createdAt": "2026-02-24T14:22:01.000Z"
    }
  ],
  "hasMore": true,
  "nextCursor": "MjAyNi0wMi0yNFQxNDoyMjowMC4wMDBa"

```

To fetch the next page, pass `nextCursor` as `afterCursor` in the next call.

## X Data

### search-tweets

Search for tweets matching a query. **Consumes API usage credits.**

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | Search query (supports X search syntax) |

**Quota:** Consumes usage credits (counts toward your monthly quota)

**Example:**

> **Prompt:** "Search for recent tweets about SpaceX Starship launch"

```json Tool Call
{
  "name": "search-tweets",
  "arguments": {
    "query": "SpaceX Starship launch"
  }
}
```

```json Result
{
  "tweets": [
    {
      "id": "1893456789012345678",
      "text": "Starship Flight 8 is GO for launch tomorrow morning.",
      "authorUsername": "elonmusk",
      "authorName": "Elon Musk",

        {
          "mediaUrl": "https://pbs.twimg.com/media/starship.jpg",

    {
      "id": "1893456789012345679",

      "authorUsername": "SpaceX",
      "authorName": "SpaceX",
      "createdAt": "2026-02-24T15:00:00.000Z"
```

**Output fields:** Each tweet may include an optional `media` array of attached media items. Each item contains `mediaUrl` (direct pbs.twimg.com URL), `type` (`photo`, `video`, or `animated_gif`), and `url` (shortened t.co URL). Omitted when the tweet has no attachments.

## Extractions

### run-extraction

Extract bulk data from X. Requires an active subscription. **Consumes API usage credits.**

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `toolType` | string | Yes | Type of extraction to run. One of: `article_extractor`, `community_extractor`, `community_moderator_explorer`, `community_post_extractor`, `community_search`, `follower_explorer`, `following_explorer`, `list_follower_explorer`, `list_member_extractor`, `list_post_extractor`, `mention_extractor`, `people_search`, `post_extractor`, `quote_extractor`, `reply_extractor`, `repost_extractor`, `space_explorer`, `thread_extractor`, `verified_follower_explorer` |
| `targetTweetId` | string | No | Tweet ID (for reply_extractor, repost_extractor, quote_extractor, thread_extractor, article_extractor) |
| `targetUsername` | string | No | X username without @ (for follower_explorer, following_explorer, verified_follower_explorer, mention_extractor, post_extractor) |
| `targetCommunityId` | string | No | Community ID (for community_extractor, community_moderator_explorer, community_post_extractor, community_search) |
| `targetListId` | string | No | X List ID (for list_member_extractor, list_post_extractor, list_follower_explorer) |
| `targetSpaceId` | string | No | X Space ID (for space_explorer) |
| `searchQuery` | string | No | Search keywords (for people_search, community_search) |

**Quota:** Consumes usage credits (counts toward your monthly quota)

**Example:**

> **Prompt:** "Extract all replies to tweet 1893704267862470862"

```json Tool Call
{
  "name": "run-extraction",
  "arguments": {
    "toolType": "reply_extractor",

}
```


## Giveaway Draws

### run-draw

Run a giveaway draw from a tweet with configurable filters. **Consumes API usage credits.**

**Input:**

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `tweetUrl` | string | Yes | - | Full URL of the giveaway tweet |
| `winnerCount` | number | No | 1 | Number of winners |
| `backupCount` | number | No | - | Number of backup winners |
| `uniqueAuthorsOnly` | boolean | No | - | Count only one entry per author |
| `mustRetweet` | boolean | No | - | Require participants to have retweeted |
| `mustFollowUsername` | string | No | - | Username participants must follow |
| `filterMinFollowers` | number | No | - | Minimum follower count |
| `filterAccountAgeDays` | number | No | - | Minimum account age in days |
| `filterLanguage` | string | No | - | Filter by language code (e.g., "en") |
| `requiredKeywords` | string[] | No | - | Keywords that must appear in replies |
| `requiredHashtags` | string[] | No | - | Hashtags that must appear in replies |
| `requiredMentions` | string[] | No | - | Usernames that must be mentioned |

**Quota:** Consumes usage credits (counts toward your monthly quota)

**Example:**

> **Prompt:** "Run a giveaway draw on https://x.com/xquik/status/1893704267862470862 with 3 winners"

```json Tool Call
{
  "name": "run-draw",
  "arguments": {
```

</Info>

### refine-tweet

Get targeted composition guidance after answering follow-up questions from `compose-tweet`. Returns goal-specific tips, example patterns, and media strategy. **Free, does not consume usage quota.**

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `topic` | string | Yes | What the tweet is about |
| `goal` | string | Yes | Optimization goal: `engagement`, `followers`, `authority`, or `conversation` |
| `tone` | string | Yes | Desired tone (e.g., casual, professional, provocative, educational) |
| `hashtags` | string | No | Hashtags to include (comma or space separated) |
| `mediaType` | string | No | Type of media to attach: `photo`, `video`, or `none` |
| `callToAction` | string | No | Desired call to action (e.g., "reply with your take") |
| `additionalContext` | string | No | Additional context about target audience or constraints |

**Example:**

> **Prompt:** "I want a casual tone, optimized for engagement, no hashtags"

```json Tool Call
{
  "name": "refine-tweet",
  "arguments": {
    "topic": "launching my new SaaS product",
    "goal": "engagement",
    "tone": "casual",
    "mediaType": "photo",
    "callToAction": "reply with your take"
```

### score-tweet

Evaluate a draft tweet against X algorithm ranking factors. Returns a pass/fail checklist covering links, hashtags, capitalization, engagement farming, CTA, length, media, and punctuation. **Free, does not consume usage quota.**

**Input:**

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `draft` | string | Yes | - | The draft tweet text to evaluate |
| `hasLink` | boolean | No | `false` | Whether a link will be attached (link card, not in tweet body) |
| `hasMedia` | boolean | No | `false` | Whether media (photo/video) will be attached |

**Example:**

> **Prompt:** "Score this draft: Just shipped dark mode for our app. What do you think?"

```json Tool Call
{
  "name": "score-tweet",
  "arguments": {
    "draft": "Just shipped dark mode for our app. What do you think?",
    "hasMedia": true
  }
}
```

```json Result
{
  "totalChecks": 8,
  "passedCount": 7,
  "topSuggestion": "Add a line break before the question to create visual separation and improve readability.",
  "checklist": [
    { "check": "No links in body", "passed": true },
    { "check": "1-2 hashtags max", "passed": true },
    { "check": "No ALL CAPS", "passed": true },
    { "check": "No engagement farming", "passed": true },
    { "check": "Has CTA or question", "passed": true },

    { "check": "Has media attached", "passed": true },
    { "check": "Line breaks for readability", "passed": false }

```

Iterate on the draft and re-score until all checks pass.
