The Xquik MCP server exposes 18 tools that AI agents can use to manage monitors, query events, search tweets, look up users and tweets, download media, check follow relationships, run extractions and draws, configure webhooks, check account status, get trending topics and radar news, manage subscriptions, and compose algorithm-optimized tweets. Connect to the MCP server at `https://xquik.com/mcp` with your API key.

## Monitoring

### monitors

Manage monitored X accounts. Use the `action` parameter to list, add, or remove monitors. Subscription required for `add`.

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `action` | `list` \| `add` \| `remove` | Yes | Action to perform |
| `username` | string | For `add` | X username without @ |
| `eventTypes` | string[] | For `add` | Event types: `tweet.new`, `tweet.reply`, `tweet.retweet`, `tweet.quote`, `follower.gained`, `follower.lost` |
| `monitorId` | string | For `remove` | Monitor ID to remove |

**Examples:**

> **Prompt:** "Show me all my active monitors"

```json Tool Call
{
  "name": "monitors",
  "arguments": { "action": "list" }
}
```

```json Result
{
  "monitors": [
    {
      "id": "7",
      "xUsername": "elonmusk",
      "eventTypes": ["tweet.new", "tweet.reply"],
      "isActive": true,
      "createdAt": "2026-02-24T10:30:00.000Z"
    }
  ]
}
```

> **Prompt:** "Start monitoring @vercel for new tweets"

```json Tool Call
{
  "name": "monitors",
  "arguments": {
    "action": "add",
    "username": "vercel",
    "eventTypes": ["tweet.new", "tweet.reply", "tweet.quote"]
  }
}
```

> **Prompt:** "Stop monitoring the account with ID 7"

```json Tool Call
{
  "name": "monitors",
  "arguments": { "action": "remove", "monitorId": "7" }
}
```

### events

**Input:**

| `monitorId` | string | No | Filter by monitor ID |

| `limit` | number | No | Results per page (1-50, default 50) |

### webhooks

Manage webhook endpoints for real-time event push. Use the `action` parameter to list, add, remove, or test webhooks.

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `action` | `list` \| `add` \| `remove` \| `test` | Yes | Action to perform |
| `url` | string | For `add` | HTTPS endpoint URL |
| `eventTypes` | string[] | For `add` | Event types to subscribe to |
| `webhookId` | string | For `remove`, `test` | Webhook ID |

**Examples:**

> **Prompt:** "Set up a webhook at https://my-server.com/events for new tweets"

```json Tool Call
{
  "name": "webhooks",
  "arguments": {
    "action": "add",
    "url": "https://my-server.com/events",
    "eventTypes": ["tweet.new", "tweet.reply"]
  }
}
```

```json Result
{
  "id": "15",
  "url": "https://my-server.com/events",
  "eventTypes": ["tweet.new", "tweet.reply"],
  "secret": "whsec_a1b2c3d4e5f6...",
  "createdAt": "2026-02-24T12:00:00.000Z"
}
```

<Warning>
  The `secret` is returned only once. Store it securely for HMAC signature verification.
</Warning>

> **Prompt:** "Send a test payload to webhook 15"

```json Tool Call
{
  "name": "webhooks",
  "arguments": { "action": "test", "webhookId": "15" }
}
```

## X Data

### search-tweets

Search X for real-time tweets by keyword, hashtag, user, or boolean query. Returns approximately 20 results. For bulk results (50+), use the `extractions` tool with `tweet_search_extractor`. Subscription required. Metered.

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `query` | string | Yes | X search query. Supports keywords, hashtags, `from:username`, `to:username`, `"exact phrases"`, and boolean operators (`OR`, `-exclude`) |

**Example:**

> **Prompt:** "What are people saying about TypeScript on X right now?"

```json Tool Call
{
  "name": "search-tweets",
  "arguments": { "query": "TypeScript performance" }
}
```

```json Result
{
  "tweets": [
    {
      "id": "1893456789012345678",

      "authorUsername": "typescript",
      "authorName": "TypeScript",
      "createdAt": "2026-02-24T14:22:00.000Z"
    }
  ]
}
```

## Extractions & Draws

### extractions

Bulk data extraction from X using 20 tool types. Use the `action` parameter to estimate costs, run jobs, list history, or get results. Always call `estimate` before `run` to preview costs. Subscription required for `run` and `estimate`. Metered (run only).

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `action` | `estimate` \| `run` \| `list` \| `get` | Yes | Action to perform |
| `toolType` | string | For `run`, `estimate` | One of the [20 extraction tool types](/guides/extraction-workflow#tool-types-reference) |
| `id` | string | For `get` | Extraction job ID |
| `targetTweetId` | string | When tool requires tweet target | Target tweet ID |
| `targetUsername` | string | When tool requires username target | Target username |
| `targetCommunityId` | string | When tool requires community target | Target community ID |
| `targetListId` | string | When tool requires list target | Target list ID |
| `targetSpaceId` | string | When tool requires Space target | Target Space ID |
| `searchQuery` | string | For search-based tools | Search query |
| `resultsLimit` | number | No | Max results to extract or estimate |
| `status` | string | For `list` | Filter by status: `completed`, `failed`, `running` |
| `limit` | number | For `list`, `get` | Results per page |
| `afterCursor` | string | For `list`, `get` | Pagination cursor |

**Examples:**

> **Prompt:** "How much would it cost to extract all replies to this tweet?"

```json Tool Call
{
  "name": "extractions",
  "arguments": {
    "action": "estimate",
    "toolType": "reply_extractor",
    "targetTweetId": "1893704267862470862"
  }
}
```

```json Result
{
  "allowed": true,
  "source": "replyCount",
  "estimatedResults": 150,
  "usagePercent": 45,
  "projectedPercent": 78
}
```

> **Prompt:** "Extract all replies to this tweet"

```json Tool Call
{
  "name": "extractions",
  "arguments": {
    "action": "run",
    "toolType": "reply_extractor",
    "targetTweetId": "1893704267862470862"
  }
}
```

```json Result
{
  "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "toolType": "reply_extractor",
  "status": "running"
}
```

> **Prompt:** "Show me my last 5 extractions"

```json Tool Call
{
  "name": "extractions",
  "arguments": { "action": "list", "limit": 5 }
}
```

> **Prompt:** "Get the results of extraction a1b2c3d4"

```json Tool Call
{
  "name": "extractions",
  "arguments": {
    "action": "get",
    "id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890"
  }
```

## Tweet Composition

### compose-tweet

Compose, refine, and score tweets using X algorithm data. Use the `step` parameter to progress through the workflow. Free. Not metered.

**Steps:**
1. `compose` (default): Returns algorithm weights, content rules, penalties, and follow-up questions
2. `refine`: Returns composition guidance, example patterns, media strategy, and CTA advice
3. `score`: Evaluates a draft against 15 algorithm checks with pass/fail results

**Input:**

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `step` | `compose` \| `refine` \| `score` | No | Workflow step (default `compose`) |
| `topic` | string | For `compose`, `refine` | What the tweet is about |
| `goal` | string | No | Optimization goal: `engagement`, `followers`, `authority`, `conversation` |
| `styleUsername` | string | No | Cached style username for voice reference (`compose`) |
| `tone` | string | For `refine` | Desired tone |
| `mediaType` | string | For `refine` | `photo`, `video`, or `none` |
| `callToAction` | string | For `refine` | Desired CTA |
| `additionalContext` | string | For `refine` | Target audience context |
| `draft` | string | For `score` | Tweet text to evaluate |
| `hasMedia` | boolean | For `score` | Whether media is attached |
| `hasLink` | boolean | For `score` | Whether a link card is attached |

**Examples:**

> **Prompt:** "Help me write a tweet about launching my new product"

```json Tool Call
{
  "name": "compose-tweet",
  "arguments": {
    "step": "compose",
    "topic": "launching my new SaaS product",
    "goal": "engagement"
  }
}
```

```json Result
{
  "signals": ["Reply-worthy content gets 2x algorithmic boost", "Tweets with media receive 35% more impressions"],
  "rules": ["Keep under 280 characters", "Front-load the hook in the first line"],
  "penalties": ["External links reduce reach by ~50%", "Hashtag stuffing (3+) triggers spam filters"],
  "followUpQuestions": [
    "What tone do you want? (casual, professional, bold)",
    "Will you attach media (photo, video, none)?",

  ]
}
```

> **Prompt:** "Score this tweet: Just shipped v2.0 of our API."

```json Tool Call
{
  "name": "compose-tweet",

    "step": "score",
    "draft": "Just shipped v2.0 of our API. What do you think?",
    "hasMedia": true
  }
}
```

```json Result

  "totalChecks": 8,
  "passedCount": 6,
  "topSuggestion": "Add a specific detail or metric to the first line for a stronger hook.",
  "checklist": [
    { "check": "Hook in first line", "passed": false },
    { "check": "Under 280 characters", "passed": true },
    { "check": "Has media", "passed": true },
    { "check": "Reply-worthy CTA", "passed": true },
    { "check": "No external links", "passed": true },
    { "check": "No hashtag stuffing", "passed": true },
    { "check": "Conversational tone", "passed": true },
    { "check": "Specific/concrete content", "passed": false }
  ]
```
