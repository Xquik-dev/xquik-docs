---
title: Create Webhook
description: Register an HTTPS endpoint to receive real-time event notifications
---

## Request

```
POST /api/v1/webhooks
```

### Headers

| Header | Required | Description |
|--------|----------|-------------|
| `x-api-key` | Yes | Your API key |
| `Content-Type` | Yes | `application/json` |

### Body

```json
{
  "url": "https://your-server.com/webhook",
  "eventTypes": ["tweet.new", "tweet.reply"]
}
```

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `url` | string | Yes | HTTPS endpoint URL |
| `eventTypes` | string[] | Yes | At least 1 event type |

<Warning>
  The URL **must** use HTTPS. HTTP endpoints are rejected.
</Warning>

## Response

<Tabs>
  <Tab title="201 Created">
    ```json
    {
      "id": "15",
      "url": "https://your-server.com/webhook",
      "eventTypes": ["tweet.new", "tweet.reply"],
      "secret": "a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2c3d4e5f6a1b2",
      "createdAt": "2026-02-24T10:30:00.000Z"
    }
    ```

    <Warning>
      The `secret` (32-byte hex) is returned **only once**. Store it securely. You need it to [verify signatures](/webhooks/verification).
    </Warning>
  </Tab>
  <Tab title="400 Invalid Input">
    ```json
    { "error": "invalid_input" }
    ```
  </Tab>
</Tabs>
