## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key. Generate one from the [dashboard](/api-reference/api-keys/create).
</ParamField>

## Response

<Tabs>
  <Tab title="200 OK">
    <ResponseField name="plan" type="string">
      Subscription status. `"active"` or `"inactive"`.
    </ResponseField>
    <ResponseField name="monitorsAllowed" type="number">
      Maximum monitors your plan allows.
    </ResponseField>
    <ResponseField name="monitorsUsed" type="number">
      Number of currently active monitors.
    </ResponseField>
    <ResponseField name="currentPeriod" type="object">
      Current billing period. Omitted if no active subscription.
      <Expandable title="currentPeriod object">
        <ResponseField name="start" type="string">
          ISO 8601 period start timestamp.
        </ResponseField>
        <ResponseField name="end" type="string">
          ISO 8601 period end timestamp.
        </ResponseField>
        <ResponseField name="capMicrocents" type="string">
          Usage cap for the current period (internal unit).
        </ResponseField>
        <ResponseField name="usedMicrocents" type="string">
          Usage consumed in the current period (internal unit).
        </ResponseField>
      </Expandable>
    </ResponseField>

    ```json
    {
      "plan": "active",
      "monitorsAllowed": 1,
      "monitorsUsed": 0,
      "currentPeriod": {
        "start": "2026-02-01T00:00:00.000Z",
        "end": "2026-03-01T00:00:00.000Z",
        "capMicrocents": "10000000",
        "usedMicrocents": "1500000"
      }
    }
    ```
  </Tab>
  <Tab title="401 Unauthenticated">
    ```json
    { "error": "unauthenticated" }
    ```
    Missing or invalid API key. Check the `x-api-key` header value.
  </Tab>
</Tabs>

## Usage Quota

Every subscriber gets a generous monthly usage quota that resets each billing period. The quota covers operations that query the X API. Management operations are unlimited.

| Operation | Monthly Limit (approx.) |
|---|---|
| Tweet searches | ~66,000 |
| User lookups | ~55,000 |
| Follower fetches | ~66,000 |
| Verified follower fetches | ~33,000 |
| Follow checks | ~66,000 |

Compare `usedMicrocents` against `capMicrocents` to track remaining capacity. When the cap is reached, metered API calls return `402`.
