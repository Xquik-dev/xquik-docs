## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key. Session cookie authentication is also supported. Generate a key from the [dashboard](https://xquik.com/dashboard).
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`.
</ParamField>

## Body

<ParamField body="toolType" type="string" required>
  Extraction tool to run. See [tool types](#tool-types) below.
</ParamField>

<ParamField body="targetTweetId" type="string">
  Tweet ID to extract from. Required for `reply_extractor`, `like_extractor`, `retweet_extractor`.
</ParamField>

<ParamField body="targetUsername" type="string">
  X username to extract from. The `@` prefix is automatically stripped if included. Required for `mention_tracker`, `follower_extractor`, `following_extractor`.
</ParamField>

<ParamField body="targetCommunityId" type="string">
  Community ID to extract members from. Required for `community_explorer`.
</ParamField>

<ParamField body="searchQuery" type="string">
  Search query string. Required for `search_results`.
</ParamField>

<ParamField body="filters" type="object">
  Optional filtering criteria applied before deduplication.
  <Expandable title="Filter properties">
    <ParamField body="filters.excludeReplies" type="boolean">
      Exclude reply tweets from results.
    </ParamField>
    <ParamField body="filters.languages" type="string[]">
      Filter by tweet language codes (e.g., `["en", "tr", "es"]`).
    </ParamField>
    <ParamField body="filters.minFollowers" type="number">
      Minimum follower count threshold.
    </ParamField>
    <ParamField body="filters.maxFollowers" type="number | null">
      Maximum follower count threshold. Set to `null` for no upper limit.
    </ParamField>
    <ParamField body="filters.followsTarget" type="boolean">
      Only include users who follow the target account.
    </ParamField>
    <ParamField body="filters.customKeywords" type="string[]">
      Filter results to tweets containing any of these keywords.
    </ParamField>
  </Expandable>
</ParamField>

<ParamField body="enrichData" type="boolean">
  Enrich results with additional user profile data (following count, post count, bio, location). Increases processing time and usage cost.
</ParamField>

<ParamField body="deduplicateBy" type="string">
  Deduplication strategy. `none` keeps all results, `username` removes duplicate users (keeps first occurrence). Default: `none`.
</ParamField>

## Tool types

Each tool type requires a specific target field.

| Tool Type | Required Field | Description |
|-----------|---------------|-------------|
| `reply_extractor` | `targetTweetId` | Extract users who replied to a tweet |
| `like_extractor` | `targetTweetId` | Extract users who liked a tweet |
| `retweet_extractor` | `targetTweetId` | Extract users who retweeted a tweet |
| `community_explorer` | `targetCommunityId` | Extract members of a community |
| `mention_tracker` | `targetUsername` | Extract users who mentioned an account |
| `follower_extractor` | `targetUsername` | Extract followers of an account |
| `following_extractor` | `targetUsername` | Extract accounts followed by a user |
| `search_results` | `searchQuery` | Extract users matching a search query |

    }

  <Tab title="400 Invalid Input">
    ```json
    { "error": "invalid_input" }
    ```
    Request body is missing or malformed. Ensure all required fields are present.
  </Tab>
  <Tab title="400 Invalid Tool Type">
    ```json
    { "error": "invalid_tool_type" }

  </Tab>

  </Tab>

  </Tab>
  <Tab title="502 X API Unavailable">

    ```
    The upstream X API is temporarily unavailable. Retry with exponential backoff.
  </Tab>
