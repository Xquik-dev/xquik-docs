Trigger a test run of an automation flow. The test uses synthetic trigger data based on the flow's trigger type and executes all steps.

<CodeGroup>
  ```bash cURL
  curl -X POST https://xquik.com/api/v1/automations/new-follower-welcome/test \
    -H "x-api-key: xq_YOUR_KEY_HERE" | jq
  ```

  ```javascript Node.js
  const slug = "new-follower-welcome";
  const response = await fetch(`https://xquik.com/api/v1/automations/${slug}/test`, {
    method: "POST",
    headers: {
      "x-api-key": "xq_YOUR_KEY_HERE",
    },
  });
  const data = await response.json();
  console.log(`Run ${data.runId}: ${data.result}`);
  ```

  ```python Python
  import requests

  slug = "new-follower-welcome"
  response = requests.post(
      f"https://xquik.com/api/v1/automations/{slug}/test",
      headers={"x-api-key": "xq_YOUR_KEY_HERE"},
  )
  data = response.json()
  print(f"Run {data['runId']}: {data['result']}")
  ```

  ```go Go
  package main

  import (
      "encoding/json"
      "fmt"
      "net/http"
  )

  func main() {
      slug := "new-follower-welcome"
      req, err := http.NewRequest("POST", "https://xquik.com/api/v1/automations/"+slug+"/test", nil)
      if err != nil {
          panic(err)
      }
      req.Header.Set("x-api-key", "xq_YOUR_KEY_HERE")

      resp, err := http.DefaultClient.Do(req)
      if err != nil {
          panic(err)
      }
      defer resp.Body.Close()

      var data map[string]interface{}
      if err := json.NewDecoder(resp.Body).Decode(&data); err != nil {
          panic(err)
      }
      fmt.Printf("Run %s: %s\n", data["runId"], data["result"])
  }
  ```
</CodeGroup>

## Path parameters

<ParamField path="slug" type="string" required>
  The flow slug.
</ParamField>

## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key. Session cookie authentication is also supported. Generate a key from the [dashboard](https://xquik.com/dashboard).
</ParamField>

## Response

<Tabs>
  <Tab title="200 OK">
    <ResponseField name="status" type="string">Always `"ok"`.</ResponseField>
    <ResponseField name="result" type="string">`"completed"` if all steps succeeded, `"failed"` if any step failed.</ResponseField>
    <ResponseField name="runId" type="string">The ID of the test run. Use [Get Automation](/api-reference/automations/get) to see run details.</ResponseField>
    <ResponseField name="error" type="string">Error message when `result` is `"failed"`. Omitted on success.</ResponseField>

    ```json
    {
      "status": "ok",
      "result": "completed",
      "runId": "42"
    }
    ```
  </Tab>
  <Tab title="400 No steps">
    ```json
    { "error": "Flow has no steps." }
    ```
    The flow has no steps configured. Add at least one step before testing.
  </Tab>
  <Tab title="401 Unauthenticated">
    ```json
    { "error": "unauthenticated", "message": "Missing or invalid API key" }
    ```
    Missing or invalid API key.
  </Tab>
  <Tab title="404 Not found">
    ```json
    { "error": "not_found", "message": "Flow not found" }
    ```
    No flow exists with this slug, or it belongs to a different account.
  </Tab>
  <Tab title="429 Rate limited">
    ```json
    { "error": "rate_limited", "message": "Too many test runs. Try again later.", "retryAfter": 60 }
    ```
    Test runs are limited to 5 per minute per user. Wait and retry.
  </Tab>
</Tabs>

<Note>
  Test runs use synthetic trigger data based on the flow's trigger type. For `monitor_event` triggers, the test searches for a recent tweet to use as trigger data. For `schedule` and `webhook_inbound` triggers, the test uses placeholder data.
</Note>

<Note>
  **Related:** [Get Automation](/api-reference/automations/get) to check run history, or [Update Automation](/api-reference/automations/update) to activate the flow for live execution.
</Note>
