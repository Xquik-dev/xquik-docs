<CodeGroup>

  ```bash cURL
  curl -X DELETE https://xquik.com/api/v1/automations/new-follower-welcome/steps \
    -H "x-api-key: xq_YOUR_KEY_HERE" \
    -H "Content-Type: application/json" \
    -d '{ "stepId": "101" }' | jq
  ```

  ```javascript Node.js
  const slug = "new-follower-welcome";
  ```

## Path Parameters

<ParamField path="slug" type="string" required>
  The flow slug.
</ParamField>

## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key. Session cookie authentication is also supported. Generate a key from the [dashboard](https://xquik.com/dashboard).
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`.
</ParamField>

## Body

<ParamField body="stepId" type="string" required>
  The unique step ID to delete.
</ParamField>

## Response

<Tabs>
  <Tab title="200 OK">
    <ResponseField name="success" type="boolean">Always `true` on successful deletion.</ResponseField>

    ```json
    { "success": true }
    ```
  </Tab>
  <Tab title="400 Invalid Input">
    ```json
    { "error": "invalid_input", "message": "Missing stepId" }
    ```
    Missing `stepId` in request body.
  </Tab>
  <Tab title="401 Unauthenticated">
    ```json
    { "error": "unauthenticated", "message": "Missing or invalid API key" }
    ```
    Missing or invalid API key.
  </Tab>
  <Tab title="404 Not Found">
    ```json
    { "error": "not_found", "message": "Flow or step not found" }
    ```
    No flow exists with this slug, or the step does not belong to this flow.
  </Tab>
</Tabs>

<Info>
  Deleting the last action step from an active flow will automatically deactivate it. Re-add a step and reactivate via [Update Automation](/api-reference/automations/update).
</Info>

  **Related:** [Add Step](/api-reference/automations/add-step) to add new steps, or [Get Automation](/api-reference/automations/get) to verify the current step list.
</Note>
