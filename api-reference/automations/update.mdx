## Path Parameters

<ParamField path="slug" type="string" required>
  The flow slug.
</ParamField>

## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key. Session cookie authentication is also supported. Generate a key from the [dashboard](https://xquik.com/dashboard).
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`.
</ParamField>

## Body

<ParamField body="expectedUpdatedAt" type="string" required>
  ISO 8601 timestamp of the flow's current `updatedAt` value. Used for optimistic concurrency control. Returns 409 if the flow was modified since this timestamp.
</ParamField>

<ParamField body="name" type="string">
  Updated flow name.
</ParamField>

<ParamField body="triggerType" type="string">
  One of: `monitor_event`, `schedule`, `search`, `webhook_inbound`.
</ParamField>

<ParamField body="triggerConfig" type="object">
  Updated trigger configuration.
</ParamField>

<ParamField body="isActive" type="boolean">
  Set to `true` to activate or `false` to deactivate. Activation requires an active subscription & at least 1 action step.
</ParamField>

<ParamField body="triggerPositionX" type="integer | null">
  Canvas X coordinate for the trigger node.
</ParamField>

<ParamField body="triggerPositionY" type="integer | null">
  Canvas Y coordinate for the trigger node.
</ParamField>

## Response

<Tabs>

    <ResponseField name="slug" type="string">URL-safe flow identifier.</ResponseField>
    <ResponseField name="triggerType" type="string">Trigger type.</ResponseField>
    <ResponseField name="triggerConfig" type="object">Trigger configuration.</ResponseField>
    <ResponseField name="isActive" type="boolean">Current active status.</ResponseField>
    <ResponseField name="runCount" type="string">Total execution count.</ResponseField>

    ```json
    { "error": "activation_requires_steps", "message": "Flow must have at least one action step to activate" }
    ```

    The flow has no action steps. Add at least 1 action step via [Add Step](/api-reference/automations/add-step) before activating.
  </Tab>
  <Tab title="404 Not Found">
    ```json

    ```

  </Tab>
  <Tab title="409 Conflict">
    ```json
    { "error": "conflict", "message": "Flow was modified since expectedUpdatedAt" }
    ```
    The flow was updated by another request. Fetch the latest version with [Get Automation](/api-reference/automations/get) and retry.
  </Tab>

<Info>
  Use the `X-Flow-Updated-At` response header as the `expectedUpdatedAt` value in your next update request. This prevents concurrent edits from overwriting each other.
</Info>
