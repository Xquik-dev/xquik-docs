## Path Parameters

<ParamField path="slug" type="string" required>
  The flow slug.
</ParamField>

## Headers

<ParamField header="x-api-key" type="string" required>
  Your API key. Session cookie authentication is also supported. Generate a key from the [dashboard](https://xquik.com/dashboard).
</ParamField>

<ParamField header="Content-Type" type="string" required>
  Must be `application/json`.
</ParamField>

## Body

<ParamField body="positions" type="object[]" required>
  Array of position updates. Maximum 10 items per request.
</ParamField>

<ParamField body="positions[].stepId" type="string" required>
  The step ID to reposition.
</ParamField>

<ParamField body="positions[].positionX" type="integer" required>
  New canvas X coordinate.
</ParamField>

<ParamField body="positions[].positionY" type="integer" required>
  New canvas Y coordinate.
</ParamField>

## Response

<Tabs>
  <Tab title="200 OK">
    <ResponseField name="success" type="boolean">Always `true` on successful update.</ResponseField>

    ```json
    { "success": true }
    ```
  </Tab>
  <Tab title="400 Invalid Input">
    ```json
    { "error": "invalid_input", "message": "Invalid positions array or exceeds 10 items" }
    ```
    Empty `positions` array, missing fields, or more than 10 items.
  </Tab>
  <Tab title="401 Unauthenticated">
    ```json
    { "error": "unauthenticated", "message": "Missing or invalid API key" }
    ```
    Missing or invalid API key.
  </Tab>
  <Tab title="404 Not Found">
    ```json
    { "error": "not_found", "message": "Flow not found" }
    ```
    No flow exists with this slug, or it belongs to a different account.
  </Tab>
</Tabs>

<Info>
  Position updates are cosmetic only. They affect the canvas layout but do not change execution order. Use the `position` field in [Add Step](/api-reference/automations/add-step) or [Update Step](/api-reference/automations/update-step) to control execution order.
</Info>

</Note>
