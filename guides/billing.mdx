Xquik uses a flat subscription model with a monthly usage quota. One monthly fee, no surprises.

## Subscription

| | |
|---|---|
| **Base plan** | $20/month |
| **Included monitors** | 1 |
| **Additional monitors** | $5/month each |
| **Payment** | Credit/debit card |

Subscribe from the [dashboard](https://xquik.com). Your subscription activates immediately and renews monthly. An active subscription is required for all API access.

## Monthly Quota

Every subscription includes a monthly usage quota that resets each billing period. All metered operations deduct from a single shared pool. There are no separate buckets per operation type.

When the quota is reached, metered API calls return `402 Payment Required`. Non-metered calls (monitors, webhooks, account, trends) continue to work normally.

<Warning>
  Metered calls are rejected once the included allowance is consumed. Enable [extra usage](#extra-usage) from the dashboard to continue making metered calls beyond the included allowance.
</Warning>

## Per-Operation Costs

Each metered operation deducts from your monthly quota at the following rates:

| Operation | Cost per call |
|-----------|---------------|
| Tweet search | $0.003 |
| User profile | $0.0036 |
| Follower fetch | $0.003 |
| Verified follower fetch | $0.006 |
| Follow check | $0.02 |
| Media download | $0.003 |
| Article extraction | $0.02 |

### Write operations

Write actions (posting tweets, liking, retweeting, following, sending DMs, profile updates, media uploads, and community actions) are metered. Per-operation rates are visible in your [dashboard](https://xquik.com).

Extractions and draws consume quota proportional to the number of results processed. Use the [estimate endpoint](/api-reference/extractions/estimate) or the `extractions` MCP tool with `action="estimate"` to preview costs before running large jobs.

<Info>
  **Free operations:** the following do not consume any quota: monitors, webhooks, integrations, account status, [radar](/api-reference/radar/list), extraction history, draw history, cost estimates, tweet composition (compose, refine, score), style cache management, drafts, and X account management.
</Info>

## Extra Usage

If you need more capacity beyond the included monthly allowance, enable extra usage from the [dashboard subscription page](https://xquik.com/subscription).

When extra usage is enabled:
- Metered calls continue beyond the included allowance instead of returning `402`
- Overage is billed separately through Stripe
- A spending limit caps how much overage you can accumulate before the next invoice

### Spending Limits

Your spending limit starts at $5 and increases automatically as you pay overage invoices:

| Tier | Spending Limit |
|------|---------------|
| 1 | $5 |
| 2 | $7 |
| 3 | $10 |
| 4 | $15 |
| 5 | $25 |

Each successful overage payment advances you to the next tier. The limit resets to your current tier at each billing period.

<Warning>
  If an overage payment fails, extra usage freezes until the balance is resolved. Manage your payment method from the dashboard to resume.
</Warning>

<Tip>
  You can disable extra usage anytime from the dashboard. When disabled, metered calls return `402` once the included allowance is consumed.
</Tip>

## Checking Usage

Call `GET /api/v1/account` to see your current period usage:

```bash
curl -s https://xquik.com/api/v1/account \
 -H "x-api-key: xq_YOUR_KEY_HERE" | jq
```

**Response:**

```json
{
  "plan": "active",
  "monitorsAllowed": 1,
  "monitorsUsed": 1,
  "currentPeriod": {
    "start": "2026-02-01T00:00:00.000Z",
    "end": "2026-03-01T00:00:00.000Z",
    "usagePercent": 45
  }
}
```

The `usagePercent` field shows how much of your monthly quota has been consumed (0-100). At 100%, metered calls are rejected.

<Tip>
  Poll the account endpoint periodically to track usage. Build alerts when `usagePercent` exceeds 80 to avoid hitting the limit unexpectedly.
</Tip>
